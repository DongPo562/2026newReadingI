# 任务：增加播放暂停功能和优化启动响应速度

## 背景
参考 project.md 中的现有架构和 floating_ui.py 中的播放逻辑

## 需求1：播放暂停功能

### 当前问题
- 点击播放按钮后音频开始播放，按钮变为"II"
- 再次点击时音频重新播放，而非暂停

### 实现要求
1. 在AudioPlayer类中增加状态追踪（playing/paused/stopped）
2. 修改播放按钮点击逻辑：
   - 播放中 → pause()
   - 暂停中 → resume()
   - 已停止 → play()
3. UI反馈：
   - 播放中："II" + 绿色背景
   - 暂停中："▶" + 橙色/黄色背景
   - 已停止："▶" + 浅蓝色背景
4. 模式播放兼容：Mode1和Mode2的暂停应暂停当前片段

## 需求2：优化启动响应延迟

### 当前问题
- 程序启动后首次悬浮到绿球需等待几秒才展开列表

### 实现要求
1. 在FloatingBall初始化时预加载audio文件夹数据
2. 预创建ListPanel UI组件（隐藏状态）
3. 文件扫描放到后台线程
4. 添加启动日志监测耗时点
